#●課題内容  
・作成物
　扉の先にいる生き物を当てるゲーム

・遊び方
　１．「START」ボタンを押すと「生き物の泣き声」が流れる
　　　かつ、各「QUESTION」ボタンに「回答候補」が入力される
　　　「START」ボタンを押すと問題がランダムで変更される

　２．「回答候補」が入力された「QUESTION」ボタンの中から、
　　　泣き声と一致する生き物を選択する

　３．「生き物の泣き声」をもう一度聞きたい場合、「Re:VOICE」ボタンを押す

　４．回答を選択すると「ファイナルアンサー？」という音声が流れる
　
　５．「マイク使用許諾」のポップが表示されるので、「許可」ボタンを押し、
　　　本当にこの回答で良い場合は「ファイナルアンサー」と音声入力する

　６．正解すると扉が開く

　７．扉をクリックすると、扉が閉じ初期化される
　
　※補足：タイトルフォントいついて
　　　　　フリーフォントを使用しているため、ローカルにインストールが必要
　　　　　インストールしていない場合、デフォルトフォントになる
　　　　　https://jp.ffonts.net/Heartland-Regular.font


#●工夫した点・こだわった点  
・扉の開閉
扉がイベント操作によって、開け閉めできるようにしました。
（ドアを開けるときは「ボタン操作」、ドアを閉めるときは「ドア画像をクリック操作」）

扉／扉の枠／生き物の表示位置、扉の動かし方に悪戦苦闘しました。
自分の思った感じにはなりませんでしたが、それっぽくはできたかなと。


・音声入力
TV番組「クイズミリオネア」をイメージして「ファイナルアンサー」と答えると判定されるように、
Web Speech APIを使用して音声入力ができるようにしました。

・タイトルフォント
フリーのフォントを使用することに挑戦。
柔らかい雰囲気にしたかったので、可愛いフォントを採用。

・ボタンデザイン
回答ボタンにグラデーションを付けてみました。
